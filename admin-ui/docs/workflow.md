# فرآیندها و Workflow ## CI/CD - استفاده از GitHub Actions برای تست و دیپلوی. - اجرای تست‌ها قبل از Merge به شاخه اصلی. - دیپلوی خودکار روی سرور پس از Build موفق. --- ## اسکریپت‌ها - `npm run lint` : بررسی کیفیت کد. - `npm run build` : ساخت پروژه. - `npm run test` : اجرای تست‌های واحد. --- ## فرآیند توسعه 1. ایجاد Branch جدید برای هر Feature. 2. Commit و Push تغییرات. 3. Pull Request و Code Review. 4. Merge به شاخه اصلی پس از تایید.
